# Diagnosis and Personalized Review Planning

Use this reference for mistake diagnosis, weak-point remediation, and study paths.

## Mistake Taxonomy

| Root Cause | Signals | Intervention |
| --- | --- | --- |
| Concept gap | Cannot define or explain the idea | concept card + teach-back |
| Condition error | Uses method where assumptions fail | boundary-case drills |
| Procedure gap | Knows concept but misses steps | worked examples + step checklist |
| Transfer gap | Solves familiar examples only | mixed-topic application questions |
| Calculation error | Setup correct, arithmetic/unit/sign wrong | slow practice + unit/sign checks |
| Proof/reasoning gap | Jumps steps or uses invalid implication | proof skeleton + missing-step drills |
| Wording mismatch | Idea mostly right but not exam wording | model-answer comparison |
| Time management | Correct but too slow | timed sets + solution templates |
| Attention error | Misreads prompt or givens | prompt annotation routine |
| Anxiety/blanking | Knowledge disappears under pressure | retrieval practice + timed simulation |

## Diagnosis Output

```markdown
## Diagnosis

- **Score estimate**: <x>/<marks>
- **What is correct**: <specific parts>
- **Lost marks**: <specific missing/wrong parts>
- **Root cause**: <taxonomy label>
- **Fix**: <targeted action>
- **Drill**: <3-5 short practice tasks>
- **Re-test**: <one follow-up question>
```

## Review Path Inputs

Ask or infer:

- Exam date and available daily study time.
- Course units included/excluded.
- Target grade or pass threshold.
- Current confidence by topic.
- Assessment format: closed/open book, multiple choice, short answer, calculation, essay, coding, oral, lab.
- Weakness history from quizzes, assignments, or user answers.

If data is missing, create a default plan with assumptions.

## Planning Heuristics

- Start with `Must Know` topics and bottleneck prerequisites.
- Alternate learning, retrieval, and exam simulation.
- Use active recall before rereading.
- Place hard mixed questions after concept repair.
- Schedule past-paper simulation near the end.
- Reserve final day for formula sheets, mistakes, and light recall, not new topics.

## Cram Plans

For 1 day:

1. Prioritize top 20 percent topics by evidence.
2. Build a one-page formula/concept sheet.
3. Do one timed mini mock.
4. Review mistakes and redo similar questions.

For 3 days:

1. Day 1: high-yield concepts and procedures.
2. Day 2: topic practice and mistake repair.
3. Day 3: timed mock, past-paper patterns, final weak spots.

For 7 days:

1. Days 1-2: build concept base and dependencies.
2. Days 3-4: question bank by topic.
3. Day 5: mixed practice.
4. Day 6: timed mock and grading.
5. Day 7: error log, formula recall, light review.

## Personalized Study Session Template

```markdown
- **Done when**: <clear completion criterion>
```
